## 一、设计原则

### 1.1 移动优先设计
- **目标设备**: iOS/Android智能手机
- **屏幕适配**: 375x812px (iPhone 13基准)
- **触摸目标**: 最小44x44px，符合平台规范
- **安全区域**: 考虑刘海屏和底部Home指示器

### 1.2 情感化设计
- **视觉层次**: 突出角色状态和情感表达
- **微交互**: 体现角色"活着"的真实感
- **延迟反馈**: 模拟真实对话的延迟感
- **缺席感知**: 通过视觉变化体现角色对玩家缺席的感知

### 1.3 平台一致性
- **iOS规范**: 遵循Human Interface Guidelines
- **Android规范**: 遵循Material Design 3
- **跨平台**: 保持核心体验一致，适当适配平台特性

## 二、整体布局结构

### 2.1 应用架构
```
App Shell
├── 底部导航栏 (44px高度)
├── 内容区域 (剩余空间)
└── 状态栏 (iOS: 44px, Android: 24px)
```

### 2.2 导航层级
```
World (地图页) ← 默认首页
├── 点击角色头像 → 对话页
    ├── 点击顶部角色信息 → 角色状态页
    └── 角色状态页 → 发消息按钮 → 对话页

Discover (时间线页)
├── 点击角色头像 → 角色状态页
└── 点击帖子 → 帖子详情页

Setting (设置页)
├── Profile → 编辑玩家信息
└── 各设置项 → 对应设置页
```

## 三、底部导航设计

### 3.1 导航栏规格
| 属性 | 数值 | 说明 |
|------|------|------|
| 高度 | 44px | 标准触摸目标高度 |
| 背景 | #FFFFFF | 白色背景，iOS有毛玻璃效果 |
| 图标 | 24x24px | 线性图标，激活状态填充 |
| 文字 | 12px | 中等字重，激活状态加粗 |

### 3.2 Tab项设计
| Tab | 图标 | 激活状态 | 文字标签 |
|-----|------|----------|----------|
| World | 地图图标 | 填充蓝色 | 世界 |
| Discover | 时间线图标 | 填充蓝色 | 发现 |
| Setting | 设置图标 | 填充蓝色 | 设置 |

### 3.3 交互反馈
- **点击效果**: 轻微缩放动画 (0.95倍)
- **切换动画**: 淡入淡出，300ms时长
- **角标显示**: 未读消息数量，红色圆点

## 四、世界地图界面

### 4.1 布局结构
```
世界地图界面
├── 状态栏 (系统状态栏)
├── 地图容器 (全屏)
│   ├── 角色头像标记 (可交互)
│   ├── 地标标记 (可交互)
│   └── 缩放控件 (右下角)
└── 底部导航栏 (固定)
```

### 4.2 角色标记设计
| 状态 | 视觉表现 | 尺寸 | 交互 |
|------|----------|------|------|
| 正常 | 圆形头像，正常亮度 | 48x48px | 点击弹出卡片 |
| 睡眠 | 头像暗淡，50%透明度 | 48x48px | 点击弹出卡片 |
| 未读 | 红色角标，数字计数 | 16x16px | 点击进入对话 |
| 情绪 | 轻微颜色变化 | - | 反映当前情绪 |

### 4.3 地图交互逻辑
- **点击角色标记**: 弹出角色信息卡片
- **长按角色标记**: 快速进入对话界面
- **双击地图**: 快速定位到所有角色
- **捏合缩放**: 标准地图缩放交互

### 4.4 角色信息卡片
```
角色信息卡片 (弹出层)
├── 头像区域 (左侧)
│   ├── 角色头像 (32x32px)
│   └── 状态指示器 (右下角)
├── 信息区域 (中部)
│   ├── 角色名称 (16px，加粗)
│   ├── 当前状态 (14px，次要文字)
│   └── 最后活动时间 (12px，辅助文字)
└── 操作区域 (右侧)
    └── 对话按钮 ("发消息")
```

## 五、对话界面

### 5.1 布局结构
```
对话界面
├── 导航栏 (自定义)
│   ├── 返回按钮 (左侧)
│   ├── 角色信息 (中部)
│   │   ├── 角色头像 (可点击)
│   │   └── 角色名称 + 状态
│   └── 更多操作 (右侧)
├── 消息列表 (可滚动)
│   ├── 系统消息 (居中)
│   ├── 角色消息 (左侧对齐)
│   └── 玩家消息 (右侧对齐)
└── 输入区域 (底部固定)
    ├── 文本输入框
    ├── 附件按钮
    └── 发送按钮
```

### 5.2 消息气泡设计
| 类型 | 背景色 | 文字色 | 最大宽度 | 圆角 |
|------|--------|--------|----------|------|
| 角色消息 | #F0F0F0 | #333333 | 70% | 18px |
| 玩家消息 | #007AFF | #FFFFFF | 70% | 18px |
| 系统消息 | 透明 | #666666 | 90% | 8px |

### 5.3 消息状态指示
- **发送中**: 旋转加载图标
- **已发送**: 对勾图标
- **已读**: 双对勾图标
- **发送失败**: 红色感叹号，可重试

### 5.4 输入区域交互
- **文本输入**: 自适应高度，最多4行
- **附件功能**: 支持图片、位置分享
- **发送逻辑**: 回车键发送，支持长按发送

### 6.1 布局结构

角色状态界面
├── 头部区域 (沉浸式)
│   ├── 背景模糊图像
│   ├── 角色头像 (大尺寸)

├── 状态卡片区域 (可滚动)

## 十二、角色创建流程

### 12.1 流程步骤
1. **空世界地图** → 引导创建第一个角色
2. **拍摄原型照片** → 选择物件作为角色原型
3. **拍摄房间角落** → 生成角色居所气质
4. **角色生成仪式** → 展示生成动画
5. **成熟度选择** → 选择角色成熟度（青年期/壮年期）
6. **草稿确认界面** → 确认角色属性，输入角色名称
7. **角色诞生** → 发送第一条消息

### 12.2 界面设计要点
- **拍摄界面**: 全屏相机预览，支持拍照和相册选择
- **生成动画**: 体现角色"诞生"的仪式感
- **确认界面**: 清晰的属性展示，包含文本输入框用于角色命名
- **成熟度选择**: 提供青年期（18-35岁）和壮年期（36-55岁）选项
- **诞生时刻**: 情感化的视觉表现，展示角色名称

### 12.3 拍摄界面设计
```
拍摄界面
├── 顶部导航栏
│   ├── 返回按钮
│   └── 标题（"拍摄原型照片"或"拍摄房间角落"）
├── 相机预览区域（全屏）
└── 底部操作栏
    ├── 相册选择按钮
    ├── 拍照按钮
    └── 闪光灯控制
```

### 12.4 拍摄功能设计
- **拍照**: 使用设备相机拍摄照片
- **相册选择**: 从相册中选择已有照片
- **实时预览**: 拍摄前实时显示相机画面
- **闪光灯**: 支持开启/关闭闪光灯
- **自动对焦**: 自动对焦功能
- **拍摄确认**: 拍摄后显示预览，可重拍或确认

## 十三、交互逻辑规范

### 13.1 导航逻辑
- **深度限制**: 最多3级导航深度
- **返回逻辑**: 清晰的返回路径指示
- **状态保持**: 页面状态在导航间保持

### 13.2 加载状态
- **骨架屏**: 内容加载时的占位显示
- **加载动画**: 明确的进度指示
- **错误处理**: 友好的错误提示和重试机制

### 13.3 手势支持
- **滑动返回**: iOS侧滑返回，Android边缘返回
- **下拉刷新**: 时间线等列表页面的刷新
- **长按操作**: 快捷操作入口

## 十四、响应式设计

### 14.1 屏幕尺寸适配
| 设备类型 | 基准尺寸 | 缩放策略 |
|----------|----------|----------|
| 小屏手机 | 320x568px | 等比例缩放 |
| 中屏手机 | 375x812px | 设计基准 |
| 大屏手机 | 414x896px | 内容扩展 |
| 平板设备 | 768x1024px | 分栏布局 |

### 14.2 横屏适配
- **地图界面**: 保持全屏显示
- **对话界面**: 左右分栏布局
- **设置界面**: 保持列表布局

### 14.3 字体缩放
- **动态类型**: 支持系统字体大小设置
- **最小字号**: 不小于12px
- **行高适配**: 根据字号自动调整
