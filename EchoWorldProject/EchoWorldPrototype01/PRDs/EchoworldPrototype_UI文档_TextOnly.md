## 1. 设计系统

### 1.1 整体视觉基调

- **色系**：浅色系，主背景为暖白（`#FAF8F5`），卡片为米色（`#F2EDE6`），有温度感而非冷白。
- **强调色**：棕金色（`#8B6F47`）用于主按钮、激活态、已读状态、角色图标边框等所有强调元素。
- **克制留白**：信息密度适中，卡片之间有明确呼吸感，不拥挤。
- **非即时通讯感**：消息状态只用简单图标（对勾），不显示「正在输入…」，不强调实时性，回复有自然延迟。
- **无警告色原则**：失踪与缺席状态以透明度降低、灰度化表达，不使用红色告警。
- **触摸目标**：所有可点击元素最小 44×44pt。

### 1.2 色彩分层

| 层级 | 色值 | 用途 |
|---|---|---|
| 主背景 | `#FAF8F5` | 页面底色 |
| 卡片背景 | `#F2EDE6` | 输入框、内容卡片 |
| 浮层背景 | `#EDE6DC` | 嵌套卡片、标签胶囊 |
| 主文字 | `#1A1714` | 标题、正文 |
| 次级文字 | `#6B6259` | 说明、角色名、行为描述 |
| 占位文字 | `#A89D92` | 时间、地点、占位符 |
| 主强调色（棕金） | `#8B6F47` | 主按钮、激活态、已读 |
| 次强调色（浅棕金） | `#B5956A` | 次级强调 |
| 轻强调（选中背景） | `#E8DDD0` | 选中态背景 |
| 危险色 | `#9E4A3A` | Sign Out、放弃等不可逆操作 |
| 极淡边框 | `#EAE3D9` | 卡片内分割线 |
| 默认边框 | `#D9CFC4` | 输入框、卡片外框 |
| 角色消息气泡 | `#F2EDE6` | 米色 |
| 玩家消息气泡 | `#8B6F47` | 棕金色 |

### 1.3 字体层级

| 用途 | 字号 | 字重 |
|---|---|---|
| 页面大标题（登录、创建等） | 26pt | Light 300 |
| 区块标题 | 18pt | Medium 500 |
| 正文 / 按钮文字 | 15pt | Regular 400 |
| 说明 / 元信息 | 13pt | Regular 400 |
| 时间 / 标签 / 状态 | 11pt | Regular 400 |

### 1.4 间距与圆角

- 页面水平内边距：20pt
- 卡片内边距：16pt
- 元素间距：8pt（紧密）/ 16pt（常规）/ 24pt（分组）
- 按钮：高度 52pt，全圆角
- 卡片：圆角 16pt
- 输入框：高度 52pt，圆角 12pt
- 消息气泡：圆角 18pt

### 1.5 消息已读/未读规范

| 状态 | 图标 | 颜色 | 说明 |
|---|---|---|---|
| 发送中 | 旋转细圆圈 | `#C4B5A5` | 仅玩家消息 |
| 已发送 ✓ | 单对勾 | `#A89D92` | 服务端已收到 |
| 已读 ✓✓ | 双对勾 | 棕金 `#8B6F47` | 角色已读，不代表会回复 |
| 失败 | 红色感叹号 | `#C0392B` | 可点击重试 |

## 2. A 账户与启动流程

### A1 · 启动画面

**功能：** 品牌展示，引导进入注册或登录。

**布局顺序（从上到下）：**
1. 全屏暖白背景，无状态栏遮挡。
2. 中央区域：ECHOWORLD 字样（两行，serif 字体，字母间距拉大，Light 字重）+ 细分割线 + 副标语「一个数字居所」（13pt，占位色，字间距 3）。
3. 底部操作区：主按钮「开始」（棕金实心，全圆角）+ 文字链「已有账户？登录」（占位色 + 棕金色「登录」）。

**交互逻辑：**
- 点击「开始」→ 进入 A3 注册。
- 点击「登录」→ 进入 A2 登录。
- 进入时 Logo 与按钮淡入（1000ms）。

---

### A2 · 登录界面

**功能：** 密码登录或验证码登录。

**布局顺序（从上到下）：**
1. 左上角返回按钮。
2. 页面标题「登录」（26pt，serif，Light）+ 副标「欢迎回来」（13pt，占位色）。
3. 邮箱输入框。
4. 密码输入框（含右侧眼睛图标）。
5. 主按钮「登录」（棕金实心）。
6. 「或」分割线（两侧细线）。
7. 次级按钮「验证码登录」（边框款）。
8. 底部说明「忘记密码？用验证码登录即可」（11pt，占位色）。

**交互逻辑：**
- 点击验证码登录：密码框切换为「输入验证码」+ 「发送验证码」按钮（60秒冷却倒计时显示）。
- 邮箱格式错误或密码错误：输入框边框变危险色，下方行内错误提示。
- 连续 5 次密码错误：提示锁定 30 分钟。
- 登录成功：无角色 → B1；有角色 → C1。

---

### A3 · 注册界面

**功能：** 输入邮箱创建账户。

**布局顺序（从上到下）：**
1. 返回按钮。
2. 标题「创建账户」+ 副标「用邮箱开始」。
3. 邮箱输入框。
4. 主按钮「继续」→ 进入 A4。
5. 底部「已有账户？登录」文字链。

**交互逻辑：**
- 邮箱格式非法：行内错误提示，阻止跳转。
- 邮箱已注册：提示「该邮箱已有账户」+ 「去登录」跳转链接。

---

### A4 · 设置密码

**布局顺序（从上到下）：**
1. 返回按钮 + 标题「设置密码」。
2. 密码输入框（含显示/隐藏）。
3. 确认密码输入框。
4. 密码规则说明（常态显示，非错误状态：8位以上，含字母和数字）。
5. 主按钮「创建账户」。

**交互逻辑：**
- 两次输入不一致：确认框下方提示。
- 提交成功 → A5 邮箱验证等待页。

---

### A5 · 邮箱验证等待页

**布局顺序（从上到下）：**
1. 中央信封图标（极简线条）。
2. 主文案「验证邮件已发送」。
3. 说明文字（含邮箱地址高亮显示）。
4. 「重新发送」按钮（60秒冷却，冷却中显示倒计时）。
5. 「没有收到？检查垃圾邮件」（11pt，占位色）。

**交互逻辑：**
- 用户点击邮件链接 → DeepLink 唤起 App → 跳转 A6。
- 链接过期（24小时）→ 提示重新发送。

---

### A6 · 玩家身份设置

**功能：** 设置在角色世界中的玩家身份（名字、头像、性别）。

**布局顺序（从上到下）：**
1. 标题「告诉角色你是谁」（serif，仪式感）+ 副标「他们会记住这个名字」（占位色）。
2. 头像选取区：圆形占位图 + 加号，点击后 ActionSheet 展开（拍照 / 相册 / 使用默认）。
3. 名字输入框（标注「必填 *」，右侧字符计数）。
4. 性别单选：「男 / 女 / 其他 / 不透露」横排胶囊，选中态棕金边框。
5. 主按钮「进入世界」，名字为空时置灰。

**交互逻辑：**
- 性别可跳过，默认「不透露」。
- 完成 → B1（首次）。后续可在 Setting → Profile 修改。

## 3. B 角色创建流程

> 共 5 步。顶部步骤进度指示器（小圆点 + 细连线，当前步棕金高亮）。任意步骤退出后下次从中断处继续。

### B1 · 空世界地图引导

**布局顺序：**
1. 全屏地图背景（MVP：暖米色渐变色块 + 城市名）。
2. 中央浮层提示卡片：「世界在等待」+ 副文 + 「开始创建」主按钮。

**交互逻辑：**
- 点击「开始创建」→ B2。地图本身不可交互。

---

### B2 · 拍摄原型照片（第 1 步）

**功能：** 拍摄角色原型（宠物、植物、玩具等，暂不支持人物）。

**布局顺序（从上到下）：**
1. 关闭按钮（×）+ 步骤进度指示器（第 1 步高亮）。
2. 标题「拍下它」+ 副标「让它成为某个人的起点」。
3. 取景框（圆角矩形，四角棕金装饰线，非完整边框）。
4. 取景框下方说明「宠物 · 植物 · 玩具 · 物品」（11pt，占位色）。
5. 底部等宽两按钮：「📷 相机」+「从相册选取」（边框款）。

**交互逻辑：**
- 选取照片后自动检测人脸：检测到 → 提示「暂不支持以人物照片创建角色」，阻止继续。
- 图像模糊 → 提示重新拍摄，提供手动输入备用入口。
- 上传成功 → B3。

---

### B3 · 拍摄房间角落（第 2 步）

**功能：** 拍摄房间一角，推导居所气质与技能预设。

**布局顺序：** 与 B2 结构一致，标题改为「拍下你的一个角落」+ 副标「让他有个地方可以回来」。

**交互逻辑：** 与 B2 相同，无人脸检测。上传成功 → B4。

---

### B4 · 角色生成仪式（第 3 步）

**功能：** 等待系统生成角色原型，传递仪式感。

**布局顺序：**
1. 全屏暖白背景。
2. 中央：多个棕金光点随机漂浮（无规律循环动画，有大小与透明度变化）。
3. 中央略下方，两行 serif 文案：「他正在从那些照片里」+「生长出来」（次级色，Light 字重）。
4. **无进度条、无百分比、无 Loading 圆圈。**

**交互逻辑：**
- 页面无可点击元素。后台每 3 秒轮询生成结果。
- 完成 → B5（光点淡出，下页淡入）。超时或失败 → 提示重试。

---

### B5 · 成熟度选择（第 4 步）

**功能：** 选择角色的起点生命阶段。

**布局顺序（从上到下）：**
1. 返回按钮 + 步骤进度（第 4 步）。
2. 标题「他现在几岁」+ 副标「这是他来到这里时的年纪」。
3. 选项卡片纵向排列，每张含：阶段名 + 年龄范围 + 一句性格特征描述。选中态：棕金边框；未选态：默认边框。
4. 主按钮「继续」，未选择时置灰。

**MVP 说明：** 仅展示「青年期（18-35岁）」与「壮年期（36-55岁）」，其余两项灰色标注「即将开放」不可选。

**交互逻辑：** 单选，点击即选中。点击「继续」→ B6。

---

### B6 · 草稿确认界面（第 5 步）

**功能：** 展示生成草稿，玩家确认名字（必填），其余可选调整。

**布局顺序（从上到下，可滚动）：**
1. 返回按钮 + 步骤进度（第 5 步）。
2. 头像预览（圆形）+ 名字输入框（右侧，标注「必填 *」）。
3. 年龄行：数值 + 左右微调按钮（根据所选成熟度动态限制：青年期 18-35岁，壮年期 36-55岁）。
4. 性别行：「男 / 女 / 中性 / 未知」四选一横排单选。
5. 背景区块（只读）：系统生成一句话背景，次级色。
6. 性格区块（只读预览）：四维度滑块仅展示，不可拖动。
7. 兴趣标签区块（只读）：横排胶囊标签。
8. 出生城市行（只读）：系统根据拍摄位置推导。
9. 底部主按钮「就是她/他了」，名字为空时置灰。

**交互逻辑：**
- 点击确认 → 调用创建接口 → 成功后进入 C1，角色出现在居所并发出第一条消息。
- **年龄限制**：年龄微调范围根据所选成熟度自动调整，无法超出限制范围。
- **草稿重置**：点击创建新角色按钮后自动重置草稿状态，确保从第一步开始。

## 4. C 世界地图

### C1 · 世界地图主页

**功能：** App 主界面默认视图，展示角色位置与状态。

**布局层次（从底到顶叠加）：**
1. **地图底层**：暖米色渐变背景 + 极简网格线纹理（浅棕灰），顶部显示城市名（小字，字间距，占位色）。MVP 不接入真实地图 API。
2. **地标点层**：预置地标以小圆圈标注，旁边地标类型文字（「咖啡馆」等），可点击触发地标卡片。
3. **角色图标层**：圆形头像图标，米色底 + 棕金边框。图标下方角色名。状态细节：
   - 正常：右上角状态点（在家=暖绿，外出=棕金，旅途中=灰）
   - 睡眠：图标透明度 60%，边框变浅棕灰，无状态点
   - 未读消息：右上角红色数字角标
   - 放弃倒计时中：边框变危险暗红，透明度 75%
   - 已失踪：图标灰度化，透明度 35%，消失于地图
4. **底部 Tab Bar**：World / Discover / Setting，高度 84pt，含安全区。

**交互逻辑：**
- 点击角色图标 → 原地弹出角色卡片浮层（C1.1）。
- 点击地标点 → 弹出地标卡片（C1.2）。
- 点击地图空白处 → 关闭已打开的卡片。
- 地图每 60 秒静默刷新角色状态。

---

### C1.1 · 角色卡片弹出层

**布局顺序（卡片内从上到下）：**
1. 角色头像（36pt）+ 角色名 + 当前行为描述（次级色，如「在咖啡馆，翻着什么，偶尔抬头」）。
2. 细分割线。
3. 城市名（左对齐）+ 角色当地时间（右对齐，占位色）。
4. 两个等宽按钮：「发消息」（棕金实心）+ 「查看状态」（边框款）。

**交互逻辑：**
- 卡片弹出时地图加半透明暖白遮罩。从底部上滑淡入（200ms）。
- 「发消息」→ D1；「查看状态」→ E1；点击卡片外区域 → 关闭。

---

### C1.2 · 地标卡片

**布局顺序：**
1. 地标名称（主色）+ 地标类型（次级色，小字）。
2. 若角色在此：头像 + 「{名字}在这里」+ 当前行为描述。
3. 若无角色：「目前没有人在这里」（占位色）。

## 5. D 对话界面

### D1 · 聊天主界面

**功能：** 玩家与角色全部对话历史，发送消息入口。

**布局顺序（从上到下）：**
1. **Header**（固定）：左侧返回 + 角色头像（可点击）+ 角色名 + 当前行为状态 + 右侧城市名与当地时间。
2. **消息列表**（可滚动，从旧到新）：
   - 角色消息：气泡靠左，米色背景（`#F2EDE6`），棕黑文字，右下角时间 + 已读状态图标。
   - 玩家消息：气泡靠右，棕金背景（`#8B6F47`），暖白文字，右下角时间 + 已读状态图标。
   - 系统状态提示（如「角色正在睡觉中」）：居中，11pt，占位色，无气泡。
3. **底部输入区**（固定，键盘弹起时上移）：
   - 中央多行输入框（米色背景，圆角 22pt）。
   - 右侧发送按钮（圆形，棕金背景），有内容时高亮，无内容时置灰。

**交互逻辑：**
- 打开时滚动到最新消息。上滑触发加载更早消息（分页）。
- 发送后玩家消息只显示一条，避免重复显示。
- 不显示「正在输入…」，角色回复直接出现，并触发推送通知。
- 点击 Header 头像 → E1 角色状态页。

---

### D2 · 放弃倒计时状态

**与 D1 的差异：**
1. Header 头像边框变为危险暗红色，透明度略降。
2. 输入框变为只读，显示提示文字「发一条消息，就是你回来了」（占位色，居中）。
3. 发送按钮不可点击；点击输入框区域 → 弹出确认提示：「发送消息即表示回来，倒计时解除」+ 「继续发送」+「取消」。
4. 消息列表正常可浏览，角色仍可发消息。

---

### D3 · 已失踪状态

**与 D1 的差异：**
1. Header 头像灰度化，透明度降低，边框浅棕灰。
2. 底部输入区整体消失，替换为静态文字「{角色名}已失踪于 {日期}」（次级色，居中）。
3. 消息列表完整保留，可向上翻至最初一条。列表底部有轻渐变遮罩（视觉上「到这里停止了」）。

---

## 6. E 角色状态页

**功能：** 查看角色当前位置、行为、情绪与今日简况，支持删除角色操作。

**布局顺序（从上到下，可滚动）：**
1. **Header**：返回按钮 + 页面标题「{角色名}的此刻」。
2. **位置与行为卡片**：
   - 位置行：定位图标 + 地标类型名（主色）+ 城市 · 具体名称 · 角色当地时间（次级色）。
   - 旅途中时显示「在去 {目的地} 的路上」。
   - 细分割线。
   - 当前行为自然语言描述（次级色）。
3. **精力与情绪卡片**：
   - 精力行：「精力」标签 + 模糊进度条（无数值，仅视觉区间）+ 文字描述（「还好」「有些疲惫」等）。
   - 情绪行：「情绪」标签 + 文字描述（「有点平静」「心情不错」等）。
4. **今日简况区块**：
   - 小标题「今日」（12pt，占位色）。
   - 「去过」行：最多 3 个地标胶囊标签，无则显示「—」。
   - 「遇见」行：最多 2 个角色头像 + 名字，无则显示「—」。
5. **近期朋友区块**（若有）：头像 + 名字 + 关系描述（「聊得火热」等），最多 3 条。
6. **底部固定按钮区**：水平排列两个按钮
   - 「发消息」按钮（棕金实心，占 2/3 宽度）
   - 「删除角色」按钮（危险样式，红色边框，占 1/3 宽度）

**交互逻辑：**
- 点击「发消息」→ D1 对话界面
- 点击「删除角色」→ 弹出确认对话框，显示角色名称和警告信息，确认后删除角色并返回上一页

**状态特殊说明：**
- 睡眠中：位置显示「正在睡觉」+「预计 {时间} 醒来」，行为描述隐藏。
- 已失踪：显示「最后出现于 {地标}，{时间}」，精力/情绪/今日简况全部隐藏，「发消息」消失。
- 放弃倒计时中：正常显示，「发消息」改为「回来」。

## 7. F Timeline

### F1 · Timeline 主界面

**功能：** 时间倒序展示角色发布的所有 post。

**布局顺序（从上到下）：**
1. **Header**（固定）：「Discover」居中，无其他操作。
2. **筛选栏**（固定，Header 下方）：「全部」+ 每个角色单独胶囊按钮。选中态：棕金实心；未选态：边框款。支持按角色名称筛选动态，显示所有用户创建的角色按钮。
3. **Post 列表**（无限滚动，时间倒序）：每条 Post 卡片包含：
   - 卡片头部：角色头像（可点击跳转状态页）+ 角色名 + 地点（「{类型} · {城市}」）+ 发布时间（玩家时区）。
   - 若为图片 post：图片全宽展示（最大 220pt 高），圆角裁切。
   - 文字内容（若有）：次级色，14pt，不截断。
   - 操作行（分割线下）：点赞图标 + 数量；评论图标 + 数量。点赞激活态：棕金色实心心形。
4. **Tab Bar**（固定底部）。

**交互逻辑：**
- 下拉刷新，上滑无限加载。
- 点击头像 → E1 角色状态页。
- 点击卡片内容区 → F2 帖子详情页。
- 点击评论图标 → F2 并聚焦评论输入框。
- 点赞立即切换（乐观更新）。

---

### F2 · 帖子详情页

**布局顺序（从上到下）：**
1. 返回按钮 + 标题「帖子」。
2. Post 原文（与 F1 卡片布局一致，完整显示）。
3. 分割线 + 「回复」标题。
4. 回复列表（时间正序）：角色回复靠左，玩家回复靠右，与 D1 消息气泡风格一致。
5. 底部输入区（与 D1 一致，无礼物入口）。

## 8. G 设置

### G1 · 设置主页

**功能：** 玩家信息展示，角色信息管理，账户与偏好设置入口。

**布局顺序（从上到下，可滚动）：**
1. **Header**：「Setting」居中。
2. **玩家信息区**（可点击进入编辑资料）：大圆形头像（52pt）+ 玩家名字（主色）+ 性别（次级色）+ 右侧箭头。
3. **角色信息区**：显示所有已创建角色的列表，包含头像、名字、状态和「查看」按钮。
4. **数据管理分组**（标题「数据管理」，占位色小字）：
   - 清空数据 → 危险色文字，点击触发确认 ActionSheet（清空所有角色数据和帖子数据，清空后返回开始界面）
5. **账户分组**（标题「账户」，占位色小字）：
   - Profile → G2 编辑资料（箭头）
   - Change Password → 修改密码页（箭头）
   - Change Email → 修改邮箱页（箭头）
   - Sign Out → 危险色文字，无箭头，点击触发确认 ActionSheet
6. **通知分组**：Notification → G3 通知设置（箭头）。
7. **偏好分组**：Languages → 语言选择（箭头）。
8. **支持分组**：ECHOWORLD Docs、Customer Service（均为外链，箭头）。
9. **法律分组**：About Us、Privacy Policy（均为外链，箭头）。

**分组视觉规则：**
- 分组标题：12pt，占位色，左对齐，上方 24pt 间距。
- 条目之间：极淡细线分隔，不分卡片。
- **按钮对齐**：所有设置项按钮保持左对齐，确保视觉一致性。
- 清空数据：危险色（`#9E4A3A`），点击后 ActionSheet 确认：「此操作将清空所有角色数据和帖子数据，此操作不可恢复」+「确认清空」（危险色）+「取消」。清空后自动返回开始界面，用户可选择注册或登录。
- Sign Out：危险色（`#9E4A3A`），点击后 ActionSheet 确认：「退出后数据完整保留，重新登录即可恢复」+「确认退出」（危险色）+「取消」。
- 退出后清除本地 Token，跳转 A1。

---

### G2 · 编辑资料

**布局顺序（从上到下）：**
1. 返回按钮 + 标题「编辑资料」+ 右侧「保存」（有修改时棕金色高亮，无修改时占位色置灰）。
2. 头像居中 + 下方「更换头像」文字链（棕金色）→ ActionSheet（拍照 / 相册 / 恢复默认）。
3. 名字输入框（必填，右侧字符计数）。
4. 性别选择（与 A6 一致）。

**交互逻辑：**
- 任意字段修改后「保存」高亮。名字为空时点击「保存」→ 行内报错。
- 保存成功 → 返回 G1，玩家信息区即时刷新。

---

### G3 · 通知设置

**布局顺序（从上到下）：**
1. 返回 + 标题「通知」。
2. 「推送通知」行：说明文字 + Toggle 开关。关闭后下方选项置灰。
3. 「静默窗口」区块：
   - 说明文字（静默窗口内不推送，消息仍接收，窗口结束后统一发出）。
   - Toggle 开关（默认开启）。
   - 开启时展示「开始时间」行 + 「结束时间」行，点击触发系统时间选择器，自动保存。
4. 底部说明「时间以您的设备时区为准」（11pt，占位色）。

## 9. 通用组件规范

### 9.1 按钮

| 类型 | 背景 | 文字颜色 | 用途 |
|---|---|---|---|
| Primary | 棕金 `#8B6F47` | 暖白 `#FAF8F5` | 主操作 |
| Secondary | 透明 | 次级棕灰 `#6B6259` | 边框 `#D9CFC4` |
| Ghost | 透明 | 棕金 `#8B6F47` | 文字链接 |
| Danger | 透明 | 危险暗红 `#9E4A3A` | 边框 `#9E4A3A` |

- 高度 52pt，全圆角。禁用态透明度 40%。加载态文字替换为 ActivityIndicator。

### 9.2 输入框

- 高度 52pt，圆角 12pt，背景 `#F2EDE6`，边框 `#D9CFC4`。
- 聚焦：边框变棕金。错误：边框变危险色，下方显示错误说明。

### 9.3 卡片

- 背景 `#FAF8F5`（主页面内）或 `#F2EDE6`（嵌套卡片）。
- 边框 1pt `#EAE3D9`，圆角 16pt，内边距 16pt。

### 9.4 角色头像图标（地图上）

| 状态 | 边框色 | 透明度 | 灰度 |
|---|---|---|---|
| 正常 | 棕金 `#8B6F47` | 100% | 否 |
| 睡眠 | 浅棕灰 `#C4B5A5` | 60% | 否 |
| 放弃倒计时 | 危险暗红 `#9E4A3A` | 75% | 否 |
| 已失踪 | 浅棕灰 `#C4B5A5` | 35% | 是 |

### 9.5 角色状态点（图标右上角）

| 状态 | 颜色 |
|---|---|
| 在家 | 暖绿 `#7A9E8A` |
| 外出 | 棕金 `#8B6F47` |
| 旅途中 | 中性灰 `#9B9692` |
| 睡眠 | 无状态点 |

### 9.6 Tab Bar

- 高度 84pt，背景 `#FAF8F5`，顶部 1pt `#EAE3D9` 分割线。
- 激活态：棕金 `#8B6F47`；非激活态：占位色 `#A89D92`。

### 9.7 页面过渡

| 场景 | 动效 |
|---|---|
| Tab 切换 | 淡入淡出，250ms |
| Stack Push | 从右滑入，300ms，弹性曲线 |
| 底部卡片弹出 | 上滑淡入，200ms |
| 生成仪式页 | 全屏淡入，800ms |
| 角色诞生进入地图 | 全屏淡入，500ms |

### 9.8 响应式适配

| 设备 | 基准 | 策略 |
|---|---|---|
| 小屏手机（320pt宽） | 等比缩放 | 页面边距缩小至 16pt |
| 中屏手机（375pt宽） | 设计基准 | 标准布局 |
| 大屏手机（414pt宽+） | 内容扩展 | 卡片拉伸，间距增大 |
| 横屏 | 对话界面 | 左右分栏布局（后续支持） |
